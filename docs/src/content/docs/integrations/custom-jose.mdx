Use a **custom JWT** when you already issue bearer tokens (internal IdP, API gateway, or multi-service mesh) and want Arivie to verify signatures with **`jose`** rather than adopting a full auth framework.

## Setup

1. Set `DATABASE_URL`, `JWT_SECRET` (HS256 for the example; adapt for RS256 in production), and model keys in `.env`.

2. Install:

   ```bash
   pnpm add jose
   ```

3. Clients send `Authorization: Bearer <jwt>` on API requests.

## Route handler

```ts
import { jwtVerify } from "jose";
import { getArivie } from "@/lib/arivie";

export async function POST(req: Request): Promise<Response> {
  const arivie = await getArivie();
  return arivie.handler(req);
}
```

## resolveUser

```ts
import { jwtVerify } from "jose";

async function resolveUser(req: Request) {
  const auth = req.headers.get("authorization");
  const token = auth?.startsWith("Bearer ") ? auth.slice(7) : null;
  if (!token) throw new Response("Unauthorized", { status: 401 });

  const { payload } = await jwtVerify(
    token,
    new TextEncoder().encode(process.env.JWT_SECRET!),
  );
  const userId = typeof payload.sub === "string" ? payload.sub : "anonymous";

  return {
    userId,
    permissions: ["analytics:read"],
    dbRole: "arivie_reader",
  };
}
```

The verified JWT's **`sub`** claim is `userId`. Missing or invalid bearer tokens throw before the handler runs.

Wire `resolveUser` in `defineArivie({ resolveUser, … })`.
