## When to use this

Apply **cost guardrails** when you need hard ceilings on agent spend: fewer rows returned per `execute`, shorter SQL timeouts. These limits are enforced inside analytics tools — not advisory prompts.

## Architecture

![Context modes — pick preload for small catalogs to reduce browse/RAG cost](../../../assets/architecture/context-modes.svg)

Base example: [`examples/with-nextjs/`](https://github.com/openscoped/data-agent/tree/main/arivie/examples/with-nextjs).

## Config patch

Row limits and query timeouts are configured on the analytics plugin source adapters and tool layer. Pair `mode: "preload"` on the analytics plugin when your catalog is small to avoid retrieval token overhead.

Per-request token budgets and `limits` top-level config are **planned** with ADR 0001 durable execution wiring.

## Run it

```bash
cd arivie && pnpm install
pnpm --filter with-nextjs dev
curl -X POST http://localhost:3000/api/chat \
  -H 'Content-Type: application/json' \
  -d '{"messages":[{"role":"user","content":"How many customers?"}]}'
```

Stress-test with a wide `SELECT` prompt and confirm the agent respects row limits in the returned result set.
