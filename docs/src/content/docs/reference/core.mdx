`@arivie/core` is the orchestrator package. It provides the `defineArivie()` factory, plugin manifest compilation, durable sessions/runs/events, and the web-standard HTTP handler.

## Installation

```bash
pnpm add @arivie/core
```

## `defineArivie(config)`

The factory. Compiles the plugin manifest, builds one Mastra agent per `agents` entry, wires the durable runtime, and exposes `prompt`, `sessions`, `events`, and `handler`.

```ts
import { defineArivie, defineAgent } from "@arivie/core";
import { analytics } from "@arivie/plugin-analytics";
import { postgresRuntime, postgresSource } from "@arivie/plugin-postgres";
import { openai } from "@ai-sdk/openai";

const arivie = await defineArivie({
  app: { id: "acme", name: "Acme" },
  model: openai("gpt-4o-mini"),
  storage: postgresRuntime({ url: process.env.DATABASE_URL! }),
  plugins: [
    analytics({
      semanticPath: "./semantic",
      mode: "preload",
      compileMetric: true,
      sources: {
        postgres: postgresSource({
          url: process.env.DATABASE_URL!,
          readOnlyRole: "arivie_reader",
        }),
      },
    }),
  ],
  agents: {
    analyst: defineAgent({
      instructions: "Answer with concise, auditable SQL-backed analysis.",
      capabilities: ["analytics.query", "analytics.compile_metric"],
    }),
  },
  context: { root: "./semantic" },
  resolveUser: async () => ({
    userId: "u1",
    permissions: ["analytics:read"],
    dbRole: "arivie_reader",
  }),
});
```

Returns an `ArivieApp`.

## `ArivieApp`

```ts
interface ArivieApp {
  app: { id: string; name: string };
  manifest: RuntimeManifest;
  runtime: Runtime;
  memory: MemoryStorage;
  sessions: Runtime["sessions"];
  events: Runtime["events"];
  prompt(input: PromptInput): Promise<string>;
  handler: (req: Request) => Promise<Response>;
  hono: Hono;
  dispose(): Promise<void>;
}
```

### `prompt(input)`

Durable one-shot over `sessions.create` — returns the agent's terminal text:

```ts
const text = await arivie.prompt({
  agent: "analyst",
  prompt: "Revenue last week?",
  user: { userId: "u1", permissions: ["analytics:read"], dbRole: "arivie_reader" },
  session: { id: "conv-1" },
});
```

Optional `onText` / `onTool` callbacks stream incremental output during the run.

### `sessions.create(input)`

Full control with a cursor-replayable `ArivieEvent` stream:

```ts
const handle = await arivie.sessions.create({
  agent: "analyst",
  prompt: "Revenue last week?",
  user: { userId: "u1", permissions: ["analytics:read"], dbRole: "arivie_reader" },
  session: { id: "conv-1" },
});

for await (const event of handle.stream) {
  // model.delta, tool.call.started, run.completed, run.failed, …
}
```

### `handler(req)`

Web-standard request handler. Mounts durable session routes and the built-in `POST /api/chat` (Mastra `@mastra/ai-sdk` under the hood):

```ts
// Next.js App Router
export const POST = (req: Request) => arivie.handler(req);

// Bun
Bun.serve({ fetch: arivie.handler });

// Hono
app.all("*", (c) => arivie.handler(c.req.raw));
```

`POST /api/chat` accepts the AI SDK / `useChat` message protocol. `POST /sessions` accepts the durable session protocol with cursor-replayable events.

## `ArivieAppConfig`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `app` | `{ id: string, name: string }` | Yes | Instance identity (`app.id` is the owner id) |
| `model` | `LanguageModel` | Yes | Default AI SDK language model |
| `storage` | `RuntimeStorage` | Yes | Durable sessions/runs/events storage |
| `plugins` | `PluginInstance[]` | No | Domain plugins (`analytics`, channels, …) |
| `agents` | `Record<string, AgentDefinition>` | Yes | Named agents with capabilities |
| `context` | `{ root: string }` | No | File-backed context layer root |
| `resolveUser` | `(req: Request) => Promise<UserContext>` | Yes | User resolution from request |
| `memory` | Mastra storage | No | Conversation memory backing store (defaults to `.arivie/memory.db`) |

Analytics-specific options (`semanticPath`, `sources`, `compileMetric`, `mode`) live on the `analytics()` plugin — not on the top-level config.

## Analytics sources (via `@arivie/plugin-analytics`)

Sources are user-named domain databases registered on the analytics plugin. Each source gets an `execute_<name>` tool on agents with the matching capability.

```ts
import { analytics } from "@arivie/plugin-analytics";
import { postgresSource } from "@arivie/plugin-postgres";

plugins: [
  analytics({
    semanticPath: "./semantic",
    mode: "preload",
    compileMetric: true,
    sources: {
      postgres: postgresSource({
        url: process.env.DATABASE_URL!,
        readOnlyRole: "arivie_reader",
      }),
    },
  }),
],
```
