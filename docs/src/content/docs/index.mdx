## What you get

**Arivie** is a TypeScript-first agent framework built **on Mastra**. It adds the durable, pluggable application layer ā€” plugin SDK, sessions/runs/events, webhook dispatch, context layer ā€” that a generic agent toolkit deliberately omits. **Analytics** (text-to-SQL on a governed semantic layer) is the first-party flagship **plugin**, not the framework itself. One Arivie app serves **one owner**; multi-tenant SaaS deploys **N instances**.

<CardGrid stagger>
  <Card title="Plugin SDK" icon="document">
    `definePlugin` contributes tools, context schemas, channels, schedules, routes, capabilities, and permissions. Analytics is just the first one.
  </Card>
  <Card title="Durable runtime" icon="rocket">
    Sessions, runs, and a cursor-replayable event log; `arivie.prompt` for one-shots, `sessions.create` for streaming control.
  </Card>
  <Card title="Boundary" icon="setting">
    Single `app.id` per instance with owner-identity assertion against Postgres.
  </Card>
  <Card title="Triggers & channels" icon="notification">
    Webhook-driven event pipelines via `defineTrigger`, `defineChannel`, and `defineSubscription`.
  </Card>
  <Card title="Tool approval" icon="approve">
    Planned ā€” human-in-the-loop tool gating via Mastra `requireApproval`. See [ADR 0001](/adr/0001-adopt-mastra-durable-execution/).
  </Card>
  <Card title="Schedules" icon="calendar">
    Single-prompt cron config via `defineSchedule` today; multi-step Mastra workflows on the [roadmap](/adr/0001-adopt-mastra-durable-execution/).
  </Card>
