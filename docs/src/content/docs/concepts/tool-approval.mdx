Tool approval / HITL is **planned**, not yet shipped. See [ADR 0001 — adopt Mastra durable execution](/adr/0001-adopt-mastra-durable-execution/).

## Roadmap shape

When shipped, dangerous tools will use Mastra's `createTool({ requireApproval: true })`. The agent stream will emit a `tool-call-approval` suspension; the host app resumes via `agent.approveToolCall({ runId })` / `declineToolCall`. Arivie's durable executor will map suspensions to an `ApprovalRequested` event on the session stream, with `approve` / `decline` on the session API.

## Use cases (planned)

- **Destructive SQL** — require human confirmation before any `execute_<source>` call in production.
- **Regulated environments** — gate all tool calls by default, then explicitly allow safe tools like `compile_metric`.
- **Per-user policies** — vary approval by user role or permissions.

## What exists today

Hard ceilings on query cost are enforced inside analytics tools today (`rowsPerQuery`, `queryTimeoutMs`). See [Cost guardrails](/recipes/cost-guardrails/). Token budgets and tool-approval policy wiring land with ADR 0001.
