Arivie exists because teams want a **TypeScript-first agent framework they own** — durable sessions, plugins, webhook dispatch, context layer — without rebuilding Mastra primitives. **Analytics** (text-to-SQL on a governed semantic layer) is the first-party flagship **plugin**, not the framework itself.

The framework ships under **Apache-2.0**, published as **17** `@arivie/*` packages at **2.0.0**, and is **built on Mastra** — not beside it. See [ADR 0002 — Arivie vs Mastra ownership](/adr/0002-arivie-vs-mastra-ownership/).

## Built on Mastra, not around it

Every Arivie package whose responsibility overlaps a Mastra primitive **extends** that primitive's interface (REQ-1). Storage uses `@mastra/pg`; the agent uses Mastra `Agent`; MCP uses `@mastra/mcp`; web chat mounts `@mastra/ai-sdk` at `POST /api/chat`. Arivie does **not** run a parallel agent loop (REQ-2).

Spend Arivie's code only on the **spine** — durable sessions/runs/events, plugin SDK, dispatch queue, context layer. Route everything else to Mastra **leaves** (agents, memory, model, streaming, guardrails, workflows).

## Single-tenant by design

**One Arivie app = one owner** (`app.id`). User-level permissions inside an instance are **pass-through to the owner's database role**: if the configured DB role cannot `SELECT` a table, the agent cannot either. Multi-tenant SaaS embedders deploy **N instances**, one per tenant.

## v2 is production-shaped

Version **2.0.0** is a domain-neutral agent framework; analytics demoted to a first-party plugin. The production deliverable list — documentation, eval suite, MCP parity, deployment recipes, web chat — ships with **full suite green** via `pnpm -r test`.

## When to choose Arivie

Choose Arivie when you want a **pluggable agent runtime** you self-host, need HTTP (`arivie.handler` / `POST /api/chat`) and MCP on the **same** Mastra agents, and can map each customer or org to **its own deployed instance**. If you need arbitrary multi-tenant routing inside one process, deploy multiple instances instead.

For analytics specifically, add the `@arivie/plugin-analytics` plugin with a curated semantic layer and read-only Postgres role.
