title: Schedules
description: Run recurring single-prompt analyses with defineSchedule.
---

Arivie supports **scheduled prompts** through `defineSchedule` / `defineSchedules`. Each schedule is a cron expression plus a prompt string — the same agent that answers ad-hoc questions can run recurring reports on a timer.

> **Roadmap:** Multi-step Mastra workflows with durable suspend/resume are planned in [ADR 0001](/adr/0001-adopt-mastra-durable-execution/). Current `schedules` are **single-prompt** cron config — not yet Mastra multi-step workflows.

## When to use

- Daily/weekly metric recaps emailed to leadership.
- Automated anomaly alerts when a KPI crosses a threshold.
- Periodic data-quality checks against the semantic layer.

## Configuration

Schedules are an array of `ArivieSchedule` objects exported alongside your app config:

```ts
import { defineSchedules } from "@arivie/core";

export const schedules = defineSchedules([
  {
    id: "daily-recap",
    cron: "0 9 * * 1-5",
    timezone: "America/Chicago",
    prompt: "Run the daily-recap skill for yesterday and write the result to reports/daily-recap.md",
  },
]);
```

Wire schedules into your deployment's dispatch runner (see `examples/with-pos-fnb/` for a production-shaped setup).

Each schedule runs **one prompt** against the configured agent. Memory is scoped to `thread: "schedule-<id>"`, `resource: app.id`.

### Multiple schedules

```ts
const schedules = defineSchedules([
  { id: "daily-recap", cron: "0 9 * * 1-5", prompt: "Generate yesterday's revenue recap" },
  { id: "weekly-trends", cron: "0 10 * * 1", prompt: "Summarize this week's order trends" },
]);
```

Schedules share the same agents, plugins, and semantic layer as the rest of the app.

## CLI helper

```bash
pnpm dlx arivie add schedule daily-recap
```

This creates a schedule file and wires it into `arivie.config.ts`.

## Runtime

The host app is responsible for evaluating cron expressions and invoking `arivie.prompt` or `arivie.sessions.create` with the configured prompt. See `examples/with-pos-fnb/arivie.config.ts` for three production cron jobs mapped to SOP skills.
