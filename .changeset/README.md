# Changesets

Hello and welcome! This folder has been automatically generated by `@changesets/cli`, a build tool that works
with multi-package repos, or single-package repos to help you version and publish your code. You can
find the full documentation for it [in our repository](https://github.com/changesets/changesets).

We have a quick list of common questions to get you started:

## What is a changeset?

A changeset is a piece of information about changes made in a branch or commit. It holds three bits of
information:

- What we need to release (which packages)
- What version we are releasing them at (major, minor, or patch)
- A changelog entry for the released packages

## When do I add a changeset?

You should add a changeset when you've made changes that should be released. This might be when you fix a
bug, add a feature, or change something that affects consumers of the package.

## How do I add a changeset?

Run `pnpm changeset` in the root of the repository. The command will prompt you for a summary of the
changes and the packages that should be released.

## What happens when I merge a PR with a changeset?

When a PR with a changeset is merged, our release tooling will open a "Version Packages" PR that bumps the
versions of the packages and updates the changelogs. When that PR is merged, the packages will be published.
